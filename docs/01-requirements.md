> **版本**：v1.3 &nbsp;|&nbsp; **日期**：2026-03-27
> **工厂所在地**：越南

## 1. 项目背景与目标

### 1.1 行业背景

家具生产行业具有以下业务特点：

- **原材料种类繁多**：木材（实木、板材）、五金配件、油漆涂料、布艺皮革、玻璃、包装材料等
- **半成品/成品两级库存**：生产过程中存在毛坯件、组装件等半成品形态
- **产品属性多维度**：同一款家具涉及不同材质、颜色、尺寸的 SKU 组合
- **BOM（物料清单）管理刚需**：一件成品家具由数十种原材料组成，需精确管理用量和损耗率
- **批次追溯需求**：木材含水率、板材甲醛等级等需要按批次追溯
- **定制化生产常见**：非标定制单处理是日常业务

### 1.2 越南工厂场景

本系统服务于**越南的家具生产工厂**，需特别考虑以下场景：

- **供应链跨国**：原材料供应商可能来自越南本地、中国、马来西亚等地
- **多币种交易**：日常使用越南盾（VND），与中国供应商用人民币（CNY），出口用美元（USD）
- **多语言需求**：工厂管理层可能是中方/越方人员，系统需支持中文、越南语、英文三语切换
- **越南税务**：越南企业使用 MST（Mã số thuế）税号，增值税（VAT）税率与中国不同
- **越南地址格式**：省 → 市/县 → 坊/社 → 街道的层级结构
- **越南电话格式**：国际区号 +84，手机号 10 位
- **越南法定假日**：需考虑越南公历和传统节日对供应链的影响
- **进出口贸易**：跨国采购涉及关税、运费、报关等额外成本

### 1.3 项目目标

构建一套**轻量级桌面端进销存管理系统**，面向中小型家具生产企业，实现：

1. ✅ **采购全流程管理**：供应商 → 采购单 → 入库 → 应付账款
2. ✅ **销售全流程管理**：客户 → 销售单 → 出库 → 应收账款
3. ✅ **库存精准管控**：实时库存、库存预警、盘点、调拨
4. ✅ **BOM 物料清单**：成品与原材料的对应关系管理
5. ✅ **多语言国际化**：支持中文、越南语、英文三语无缝切换
6. ✅ **多币种支持**：VND / CNY / USD 多币种交易与汇率管理
7. ✅ **数据本地存储**：所有数据存储在本地 SQLite，无需联网
8. ✅ **客户端安装**：提供 Windows / macOS 安装包，一键安装使用
9. ✅ **固定打印模板**：提供固定单据模板 + 打印参数配置，适配各类单据打印
10. ✅ **关键物料轻量批次追溯**：支持批次库存、库龄分析、原单退货追溯

> **v1.0 范围边界**：系统提供业务单据、库存、报表、打印和简易应收应付辅助能力，**不替代专业财务软件，也不等同于越南法定税务/发票系统**。

### 1.4 用户画像

| 角色 | 典型场景 | 关注功能 |
|------|---------|---------|
| 老板/管理层 | 查看经营概况、利润分析 | 首页看板、报表统计 |
| 采购员 | 日常采购下单、供应商对账 | 采购管理、应付账款 |
| 销售员 | 客户报价、下单、跟踪发货 | 销售管理、应收账款 |
| 仓库管理员 | 出入库操作、盘点、库存查询 | 库存管理、出入库单据 |
| 生产主管 | 查看 BOM、评估物料需求 | BOM 管理、库存查询 |
| 越南本地员工 | 日常操作（越南语界面） | 出入库操作、库存查询 |

### 2.4 国际化（i18n）方案

系统支持三种语言，用户可在界面顶栏随时切换：

| 语言 | locale | 适用场景 |
|------|--------|----------|
| 🇨🇳 简体中文 | `zh` | 中方管理人员、与中国供应商对接 |
| 🇻🇳 越南语 | `vi` | 越南本地员工日常操作 |
| 🇺🇸 英文 | `en` | 国际化通用、出口客户沟通 |

**翻译范围**：
- 所有菜单、按钮、标签文字
- 表单校验提示
- 系统通知和弹窗
- 固定单据打印模板（根据语言切换）
- 日期/数字/货币格式根据 locale 自动适配

**不翻译的内容**：
- 用户自定义输入的业务数据（物料名称、客户名称等由用户输入）
- 单据编号（统一使用英文前缀）

### 2.5 多币种支持

| 币种 | 符号 | 适用场景 |
|------|------|----------|
| VND (越南盾) | ₫ | 本地采购、工资、水电等日常支出 |
| CNY (人民币) | ¥ | 中国供应商采购 |
| USD (美元) | $ | 出口销售、国际结算 |

**汇率管理**：
- 系统设置中维护各币种对 USD 的汇率
- 采购/销售单据支持选择币种
- 报表统计时统一折算为基准币种（默认 USD）
- 支持手动更新汇率，记录汇率历史
- `v1.0` 不处理汇兑损益（汇率变动导致的差额）；`v2.0` 将增加汇兑损益自动计算与调整功能

**价格与成本口径**：
- `v1.0` 系统默认/基准币种固定为 **USD**，仅作为系统核算基准，不在首次使用向导中开放修改
- 物料主数据中的「参考进价」「销售价格」统一按 **USD** 维护，作为系统内的基准参考价
- 供应商的实际报价和采购成交价按原币种记录在采购单、供应商-物料报价中
- BOM 成本、库存平均成本、报表汇总金额统一折算为 **USD** 进行核算和展示

**数量精度与单位规则**：
- 计量单位统一从「单位主数据」中选择，每个单位维护中/越/英名称和数量小数位
- 物料主数据维护基本单位和可选辅助单位，库存、BOM、成本核算统一以**基本单位**为准
- 业务单据保存时同时记录 `unit_id + 单位名称快照 + 换算率快照(conversion_rate_snapshot) + 基本单位数量(base_quantity)`，兼顾主数据统一和历史追溯
- 换算率快照 = 制单时辅助单位→基本单位的换算率；使用基本单位录入时 `conversion_rate_snapshot = 1`，`base_quantity = quantity`
- BOM 明细统一以基本单位录入，无需换算快照
- 数量录入、导入、展算和报表展示统一按单位配置的小数位进行四舍五入

**金额折算与舍入规则**：
- 所有原币明细金额先按**原币最小货币单位**完成行级计算与四舍五入，再参与单据头汇总
- 单据头的原币合计金额以已舍入的明细行金额求和，不允许前端按展示值重新回算
- 折扣、运费、其他费用先在原币口径分别舍入，再参与应收/应付金额计算和 USD 折算
- 所有基准币种（USD）金额统一由 Rust 服务层按统一公式计算并落库，前端仅负责展示
- 报表、Dashboard、单据详情统一复用同一套已落库金额口径，避免前后端分别计算产生差异

**标准成本与实际成本口径**：
- **标准成本**：基于 BOM 和物料参考进价得到，用于报价、标准成本分析和标准毛利
- **实际成本**：按库存移动加权平均成本在出库确认时固化，用于库存结转、实际毛利和经营分析
- 销售出库确认时，系统需同时固化**标准成本快照**与**实际成本快照**；其中标准成本快照来源于出库当时命中的 BOM 版本及其标准成本结果
- 销售毛利分析、导出报表和 Dashboard 涉及毛利口径时，统一基于出库时已落库的成本快照计算，不回查当前 BOM 重新计算历史数据
- 报表中的毛利分析必须明确区分“标准毛利”和“实际毛利”，不得混用

**本地认证与会话策略（v1.0）**：
- v1.0 为**单机单帐号**模式，系统内置唯一管理员帐号 `admin`（初始密码 `admin123`），拥有全部功能权限
- 系统首次启动默认进入**登录页**，使用内置管理员帐号登录
- 首次登录后必须立即修改密码，修改完成后方可继续初始化配置
- 登录密码仅以哈希形式保存；勾选“记住我”时，只允许将**加密后的本地会话凭证**保存到系统钥匙串/凭据管理器
- `v1.0` 不允许在配置文件或数据库中明文保存密码
- 如系统检测到初始化配置未完成，登录并完成首次改密后自动进入首次使用向导
- 改密后系统自动使旧的记住我会话失效
- 退出登录时删除本地会话凭证

## 3. 功能模块设计

### 3.1 首页看板（Dashboard）

经营数据的可视化概览页面，为管理者提供决策支持。

#### 3.1.1 KPI 指标卡片

| 指标 | 说明 | 计算方式 |
|------|------|---------|
| 今日销售额 | 当日已执行销售金额 | `SUM(今日已确认销售出库.应收金额)` |
| 本月销售额 | 当月累计已执行销售金额 | `SUM(本月已确认销售出库.应收金额)` |
| 今日采购额 | 当日已执行采购金额 | `SUM(今日已确认采购入库.应付金额)` |
| 库存预警数 | 低于安全库存的物料数量 | `COUNT(可用库存 < 安全库存的物料)` |
| 待收款总额 | 未结清的应收账款 | `SUM(应收账款.未收金额)` |
| 待付款总额 | 未结清的应付账款 | `SUM(应付账款.未付金额)` |

> **口径说明**：Dashboard 中所有金额类指标默认采用**执行口径**。销售金额基于已确认销售出库，采购金额基于已确认采购入库，并与应收/应付生成口径保持一致。

#### 3.1.2 图表区

- **近 30 天销售趋势**：折线图，展示按已确认销售出库统计的日销售执行金额变化
- **近 30 天采购趋势**：折线图，展示按已确认采购入库统计的日采购执行金额变化
- **库存分布**：饼图，按物料分类展示库存金额占比
- **热销产品 TOP 10**：条形图，按销量排名

#### 3.1.3 快捷操作

- 新建采购单、新建销售单、入库、出库的快捷入口
- 显示待处理事项（待审核单据、库存预警等）

### 3.2 基础数据管理

所有业务模块的基础数据维护。

#### 3.2.1 物料/产品管理

管理所有原材料、半成品、成品的基本信息。

**物料字段设计**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 物料编码 | string | ✅ | 系统自动生成或手动输入，唯一 |
| 物料名称 | string | ✅ | 如「白橡实木板」 |
| 物料分类 | enum | ✅ | 原材料 / 半成品 / 成品 |
| 二级分类 | ref | ✅ | 关联分类表（木材、五金、油漆等） |
| 规格型号 | string | — | 如「2440×1220×18mm」 |
| 基本单位 | ref | ✅ | 从单位主数据选择，如张、个、千克、米、立方米等 |
| 辅助单位 | ref | — | 可选的第二计量单位 |
| 换算比例 | number | — | 基本单位与辅助单位的换算关系 |
| 参考进价 | decimal | — | 默认采购参考价，统一按 USD 维护 |

### 3.4 采购管理

#### 3.4.1 采购单

**业务流程**：

```
创建采购单 → [审核] → 采购入库（可分批） → 生成应付账款
         ↘ [作废]                    ↘ 采购退货 → 冲减应付
```

**v1.0 审批与税务约定**：
- 审核权限采用简化模型：v1.0 为单帐号模式，管理员拥有全部权限（含审核/作废），无角色限制
- v1.0 **不支持反审核**；已审核单据如需修改，采用「作废后重建」方式处理
- v1.0 **不单独维护 VAT 税率/税额字段**，采购金额按业务约定口径录入；关税、报关费、保险等计入「其他费用」，税务备注写入单据备注

**v1.0 仓库粒度约定**：
- `v1.0` 采用**订单级单仓**模型：采购单头维护统一「入库仓库」
- 采购单明细保留「目标仓库」字段，但 `v1.0` 仅作为只读快照，必须与单头入库仓库一致
- 为未来升级预留：后续版本可放开明细行仓库，并按仓自动拆分生成执行入库单

### 3.5 销售管理

#### 3.5.1 销售单

**业务流程**：

```
创建销售单 → [审核] → 销售出库（可分批） → 生成应收账款
         ↘ [作废]                    ↘ 销售退货 → 冲减应收
```

**v1.0 审批与税务约定**：
- 审核权限与采购单一致：v1.0 单帐号模式，管理员拥有全部操作权限
- v1.0 **不支持反审核**；已审核销售单如需修改，采用作废后重建
- v1.0 **不单独维护 VAT 税率/税额字段**，销售金额按合同/报价口径录入；如涉及运费、关税等附加费用，计入单据对应费用字段

**v1.0 仓库粒度约定**：
- `v1.0` 采用**订单级单仓**模型：销售单头维护统一「出库仓库」
- 销售单明细保留「出库仓库」字段，但 `v1.0` 仅作为只读快照，必须与单头出库仓库一致
- 为未来升级预留：后续版本可放开明细行仓库，并按仓自动拆分生成执行出库单

**销售单头信息**：

| 字段 | 类型 | 说明 |
|------|------|------|
| 销售单号 | string | 自动生成，规则：`SO-YYYYMMDD-XXX` |
| 客户 | ref | 关联客户 |
| 销售日期 | date | |
| 交货日期 | date | |
| 出库仓库 | ref | `v1.0` 整单统一出库仓库 |
| **结算币种** | **enum** | **VND / CNY / USD，从客户默认带出** |
| **汇率** | **decimal** | **从系统汇率表取最近生效汇率自动填入，可手动修改** |
| 单据状态 | enum | 草稿 / 已审核 / 部分出库 / 已出库 / 已作废 |
| 合计金额 | computed | 明细行自动汇总（原币） |
| 合计金额(USD) | computed | 原币金额 × 汇率 |
| 折扣率(%) | number | 整单折扣率 |
| 折扣金额 | computed | |
| **运费** | **decimal** | **出口销售运费** |
| **其他费用** | **decimal** | **关税、报关费、保险等** |

| 备注 | string | |

**销售单明细**：

| 字段 | 类型 | 说明 |
|------|------|------|
| 物料/产品 | ref | 关联物料（通常为成品） |
| 规格型号 | string | 自动带出 |
| 单位 | ref | 来自单位主数据，保存名称快照 |
| 换算率快照 | number | 辅助单位→基本单位换算率，基本单位时为 1 |
| 销售数量 | number | |
| 基本单位数量 | computed | `销售数量 × 换算率快照` |
| 单价 | decimal | |
| 折扣率(%) | number | 行折扣 |
| 金额 | computed | `数量 × 单价 × (1 - 折扣率)` |
| 已出库数量 | number | 系统自动记录 |

**折扣叠加规则**：行级折扣先行计算，再对汇总小计应用整单折扣。公式：`应收金额 = Σ(行金额 × (1 - 行折扣率)) × (1 - 整单折扣率) + 运费 + 其他费用`。

**功能要求**：
- 销售单的新建、编辑、审核、作废
- 选择客户后自动带出默认折扣率和收货地址
- 修改单头出库仓库时，未出库明细的仓库快照同步更新
- 选择产品时显示**当前可用库存**，库存不足时给出提示；当可用库存为 **零** 时强制拦截，禁止创建出库单
- 支持按销售单号、客户、日期范围、状态筛选查询
- 审核销售单时，若客户已设置信用额度，系统自动检查当前应收总额是否将超限，超限则弹窗提醒（不强制拦截，`v2.0` 可配置为拦截）
- 销售单打印
- 销售单导出为 Excel

#### 3.6.1 库存查询

**查询维度**：

| 维度 | 说明 |
|------|------|

| 按分类 | 查看某个分类下所有物料的库存 |
| 按批次 | 查看同一物料在不同批次/入库日期下的库存和库龄 |

**展示信息**：
- 物料编码、名称、规格
- 当前库存数量
- 预留库存数量
- 可用库存数量（`当前库存 - 预留库存`）
- 安全库存、最高库存
- 库存预警状态（低于安全库存标红、高于最高库存标黄，预警基于可用库存判断）
- 库存金额（数量 × 移动加权平均成本）
- 最后出入库时间
- 批次号/最早入库日期/库龄天数（对启用批次追踪的物料展示）
- 对启用批次追踪且已预留的物料，批次可用量按 `批次在库 - 批次预留` 展示

**功能要求**：
- 支持模糊搜索物料名称/编码
- 支持筛选：仓库、分类、预警状态
- 支持展开查看批次分布、库龄和近期批次流水
- 导出库存明细为 Excel
- **期初库存导入**：首次使用系统时，支持 Excel 批量导入历史库存数据（生成「其他入库」流水记录）

**期初库存导入模板**：

系统提供标准 Excel 导入模板，并说明与原始 Excel（如工厂现有进销存明细表）的字段映射关系：

| 模板列 | 说明 | 映射源示例（工厂Excel） |
|--------|------|----------------------|
| 物料编码 | 系统物料编码（必填，需先在系统中创建物料） | 品号 |
| 物料名称 | 用于辅助核对，不参与导入逻辑 | 品名 |
| 规格型号 | 用于辅助核对 | 规格 |
| 仓库编码 | 目标仓库（必填） | — |
| 期初数量 | 库存数量（必填） | 期初 |
| 单位 | 计量单位 | 单位 |
| 参考单价 | 必填，作为期初移动加权平均成本基准（USD） | — |
| 批次号 | 启用批次追踪的物料建议填写；为空时可由系统生成 | — |

> **导入逻辑**：系统根据「物料编码 + 仓库编码」匹配，为每条记录生成一笔「其他入库」类型的库存流水，直接设置库存初始值。`参考单价` 为必填，用于初始化该仓该物料的移动加权平均成本。对启用批次追踪的物料，导入时同步生成批次库存。对于工厂现有 Excel 中缺少物料编码的记录（如仅靠规格区分的配件），需先在物料管理中创建并获取系统编码后再导入。

**成本核算规则**：
- 系统采用**移动加权平均法**核算库存成本
- 每次入库后重新计算平均成本：`新平均成本 = (原库存数量 × 原平均成本 + 本次入库数量 × 本次入库单价) ÷ 新库存数量`
- 销售出库、调拨出库、盘亏等出库类动作按当前移动加权平均成本结转
- 期初库存导入的参考单价作为首次成本基准
- 定制单或已确认销售需求可生成库存预留，库存查询、预警和补货建议统一按**可用库存**口径展示
- BOM 标准成本用于报价和标准成本分析；实际毛利分析以出库时固化的实际成本快照为准

**预留与批次分配规则**：
- 定制单确认或已确认销售需求生成库存预留时，先按物料 + 仓库生成预留主记录
- 对启用批次追踪的物料，系统默认按 **FIFO** 预占批次，并记录批次分配明细
- 人工指定出库批次时，如与原预占批次不一致，系统需同步调整对应批次预留
- 批次预留分配用于支撑库存查询中的批次可用量、后续出库消耗与预留释放，不能只停留在物料汇总层

#### 3.7.3 财务生成与退货冲减规则

**应付生成口径**（按执行单生成，含费用分摊）：

1. 采购入库确认时，系统自动生成一笔应付记录
2. 应付金额 = 入库明细金额合计 + 按比例分摊的订单级费用
3. 分摊公式：
   - 分摊比例 = 本次入库明细金额合计 ÷ 采购单明细合计金额（仅明细行）
   - 分摊折扣 = 采购单折扣金额 × 分摊比例（取负）
   - 分摊运费 = 采购单运费 × 分摊比例
   - 分摊其他费用 = 采购单其他费用 × 分摊比例
   - 最终应付 = 入库明细合计 - 分摊折扣 + 分摊运费 + 分摊其他费用
4. 费用分摊结果记录在入库单头，确保与应付记录金额一致
5. 非采购入库（其他入库）不生成应付

**应收生成口径**（与应付对称）：

1. 销售出库确认时，系统自动生成一笔应收记录
2. 应收金额 = 出库明细金额合计 + 按比例分摊的订单级费用
3. 分摊逻辑与采购对称，基于销售单的折扣/运费/其他费用按出库占比分摊
4. 非销售出库（其他出库）不生成应收

**退货冲减模型**（独立调整记录，不回写原单）：

1. 采购退货确认时，生成一笔**负金额**的应付调整记录，关联原应付单和退货单
2. 销售退货确认时，生成一笔**负金额**的应收调整记录，关联原应收单和退货单
3. 调整记录类型标记为 `return_offset`，与正常入库/出库生成的记录区分
4. 供应商/客户未付/未收余额 = 正常应付/应收合计 + 退货调整合计（负数抵扣）
5. 对账单和报表统一展示正常记录和调整记录，余额自动轧差

> **设计原则**：退货不修改已确认的原始应付/应收记录，通过独立调整记录实现冲减，保证数据可追溯。

#### 3.9.6 用户与操作权限（v1.0）

**v1.0 部署模式：单机单帐号**

v1.0 定位为**单机桌面应用**，系统内置唯一管理员帐号，同时拥有管理员和操作员的全部权限，**不做角色差异和权限限制**。SQLite 数据库文件存放在本地磁盘，不通过网络共享。该模式的优势是零运维、离线可用、操作简便。

> **v2.0 演进方向**：v2.0 计划将数据库从 SQLite 迁移至 **PostgreSQL**，同时启用**多帐号 + 角色差异权限**：
> - 支持创建多个用户帐号（管理员 / 操作员），通过用户管理页面进行增删改查
> - 管理员拥有全部权限（系统设置、审核、作废、数据备份等）
> - 操作员仅拥有日常业务权限（基础资料维护、采购/销售草稿录入、出入库操作等）
> - 后续版本可进一步扩展为自定义角色和页面级/按钮级权限配置
>
> 为降低 v2.0 迁移成本，v1.0 开发阶段需遵循以下预留原则：
> - **SQL 兼容性**：避免使用 SQLite 专有语法（如 `datetime(‘now’,‘localtime’)`），优先使用 ANSI SQL 标准写法
> - **数据访问层抽象**：Rust Service 层通过统一的 Repository trait 访问数据库，隔离具体 SQL 方言
> - **连接管理抽象**：数据库连接初始化通过配置切换，不硬编码 SQLite 特定的 PRAGMA
> - **迁移脚本兼容**：DDL 脚本同时维护 SQLite 和 PostgreSQL 版本，或使用 sqlx 的跨数据库迁移能力
> - **不依赖 SQLite 特有功能**：如 `GENERATED ALWAYS AS ... STORED` 改为应用层计算，`AUTOINCREMENT` 预留切换为 PostgreSQL `SERIAL`/`BIGSERIAL` 的空间
> - **users 表保留 role 字段**：v1.0 仅有一条 `admin` 记录且不做权限校验，但表结构保留 `role` 字段和 `CHECK` 约束，v2.0 启用时无需改表

**v1.0 规则说明**：
- 系统仅内置一个管理员帐号 `admin`，不提供用户管理功能（无新增/编辑/删除用户）
- 管理员帐号拥有所有页面和操作的完全访问权限，前端不做权限裁剪

- 改密后历史本地会话立即失效

```
客户需求沟通 → 创建定制单 → 确认定制配置 → 定制BOM展算 → 报价确认 → 转销售单 → 生产交付
```

**定制单头信息**：

| 字段 | 类型 | 说明 |
|------|------|------|
| 定制单号 | string | 自动生成，规则：`CO-YYYYMMDD-XXX` |
| 客户 | ref | 关联客户 |
| 定制日期 | date | |
| 交付日期 | date | 承诺交付日期 |
| 结算币种 | enum | VND / CNY / USD |
| 汇率 | decimal | |
| 定制类型 | enum | 尺寸定制 / 材质定制 / 全定制 |
| 优先级 | enum | 普通 / 加急 / 特急 |
| 状态 | enum | 报价中 / 已确认 / 生产中 / 已完成 / 已取消 |
| 参考产品 | ref | 基于哪个标准产品定制（可选） |
| 定制描述 | text | 客户的定制要求详细描述 |
| 报价金额 | decimal | 定制报价（原币） |
| 报价金额(USD) | computed | 原币 × 汇率 |
| 附件路径 | string | 客户提供的设计图/参考图本地路径 |
| 备注 | string | |

**定制单状态转换规则**：

```
报价中 ──→ 已确认 ──→ 生产中 ──→ 已完成
  │            │
  └──→ 已取消  └──→ 已取消
```

| 转换 | 触发条件 | 前置校验 |
|------|---------|---------|
| 报价中 → 已确认 | 管理员点击「确认」| 定制配置和 BOM 展算已完成，报价金额 > 0 |
| 报价中 → 已取消 | 点击「取消」| 无 |
| 已确认 → 生产中 | 点击「开始生产」（自动创建生产工单）或转销售单后自动 | — |
| 已确认 → 已取消 | 点击「取消」| 如已生成预留需先释放 |
| 生产中 → 已完成 | 点击「完成生产」| — |

> 已完成和已取消为终态，不可逆转。预留在"已确认"时生成，在"出库"时消耗，在"已取消"时自动释放。

#### 3.10.2 定制配置明细

| 字段 | 类型 | 说明 |
|------|------|------|
| 配置项 | string | 如「尺寸」「材质」「颜色」「五金件」等 |
| 标准值 | string | 标准产品的默认配置 |
| 定制值 | string | 客户要求的定制配置 |
| 加价金额 | decimal | 因定制产生的加价 |
| 备注 | string | |

#### 3.10.3 功能要求

- 定制单创建与编辑，支持关联标准产品快速创建
- 定制配置管理：可逐项配置尺寸、材质、颜色、五金件等
- **定制 BOM**：自动复制标准产品 BOM，支持根据定制配置调整用料
- **定制成本核算**：根据定制 BOM 自动计算定制成本，辅助报价决策
- 定制单状态跟踪：从报价到交付的全流程状态管理
- **转销售单**：确认后一键转为销售单，携带定制信息
- **库存联动**：定制单确认后生成物料预留清单，优先占用现有库存；库存查询中同步展示预留库存与可用库存；缺口自动进入补货建议，转销售单后仍按正常出库流程扣减实际库存
- 对启用批次追踪的物料，定制单确认时默认按 FIFO 预占批次，后续出库如人工改批次需同步调整预留分配
- 定制单打印：含定制配置详情、定制 BOM
- 定制历史查询：按客户查看历史定制记录

### 3.10a 生产工单（v1.0 最简版）

v1.0 提供最简化的生产工单模块，覆盖「**领料出库 → 完工入库**」核心流程。不含排产计划、工序管理、报工和产能管理，这些高级功能留待 v2.0 迭代。

#### 3.10a.1 业务流程

```
创建工单（手动/从BOM/从定制单） → 领料出库（原材料出库） → 生产中 → 完工入库（成品/半成品入库） → 关闭
```

#### 3.10a.2 工单头信息

| 字段 | 类型 | 说明 |
|------|------|------|
| 工单编号 | string | 自动生成，规则：`WO-YYYYMMDD-XXX` |
| 关联 BOM | ref | 必填，关联 BOM 版本（自动填充物料需求清单） |
| 关联定制单 | ref | 可选，从定制单生成时自动填充 |
| 产出物料 | ref | BOM 的父项物料（成品/半成品） |
| 计划数量 | decimal | 计划生产的成品数量 |
| 已完工数量 | decimal | 实际完工入库的累计数量 |
| 状态 | enum | 草稿 / 领料中 / 生产中 / 已完工 / 已取消 |
| 计划开始日期 | date | 可选 |
| 计划完成日期 | date | 可选 |
| 实际开始日期 | date | 首次领料时自动填充 |
| 实际完成日期 | date | 完工入库时自动填充 |
| 备注 | text | |

#### 3.10a.3 工单状态转换规则

```
草稿 ──→ 领料中 ──→ 生产中 ──→ 已完工
  │                    │
  └───→ 已取消         └───→ 已取消（仅未完工时）
```

| 转换 | 触发条件 | 前置校验 |
|------|---------|---------|
| 草稿 → 领料中 | 确认领料（首次生成领料出库单） | BOM 展算物料充足或允许部分领料 |
| 草稿 → 已取消 | 点击「取消」 | 无 |
| 领料中 → 生产中 | 领料完成，点击「开始生产」 | 至少有一笔已确认的领料出库 |
| 生产中 → 已完工 | 完工入库单确认 | 已完工数量 > 0 |
| 生产中 → 已取消 | 点击「取消」 | 未完工数量需退料处理（生成退料入库） |

> 已完工和已取消为终态，不可逆转。

#### 3.10a.4 领料出库

- 系统根据 BOM 和计划生产数量自动展算每种原材料的需求量（含损耗率）
- 领料时选择领料仓库和批次（启用批次追踪的物料），生成出库单（类型：`production`）
- 支持分批领料：同一工单可多次领料，累计领料量不得超过计划需求量的 120%（防止超领）
- 领料出库确认后自动扣减原材料库存，生成库存流水（类型：`production_out`）
- 支持退料：生产剩余物料可通过退料入库归还仓库（类型：`production`，入库方向）

#### 3.10a.5 完工入库

- 录入实际完工数量，选择入库仓库，生成入库单（类型：`production`）
- 完工入库确认后自动增加成品/半成品库存，生成库存流水（类型：`production_in`）
- 支持分批完工：同一工单可多次完工入库
- 完工成本 = 实际领料成本汇总 ÷ 完工数量（简化版，v1.0 不含人工和制造费用）

#### 3.10a.6 功能要求

- 工单创建：手动创建（选择 BOM）/ 从 BOM 列表快速创建 / 从定制单「开始生产」自动生成
- 工单与定制单联动：定制单状态「已确认 → 生产中」时自动创建工单；工单完工后定制单状态变为「已完成」
- 领料需求清单：基于 BOM 展算，展示每种物料的需求量、当前库存、缺口量
- 库存检查：领料前检查物料库存是否充足，不足时提示但不阻止（允许部分领料）
- 工单进度跟踪：列表页展示工单状态、计划/实际数量、领料进度、完工进度
- 工单打印：工单信息 + 领料清单
- **v2.0 演进方向**：排产计划、工序路线、报工管理、产能管理、质检集成

### 3.11 智能补货建议

基于历史消耗数据和库存状态，系统自动生成补货建议，减少人工判断，降低缺货风险。

#### 3.11.1 补货建议生成

**触发条件**：
- 用户访问「补货建议」页面时按需计算（非后台定时任务），对比当前库存与安全库存
- 用户也可在物料列表手动点击「刷新补货建议」
- 首页看板的补货提醒卡片在每次进入首页时延迟计算

**补货建议内容**：

| 字段 | 说明 |
|------|------|
| 物料 | 需要补货的物料 |
| 当前库存 | 当前物理库存数量 |
| 预留库存 | 已被定制单/已确认需求占用的库存 |
| 可用库存 | `当前库存 - 预留库存` |
| 安全库存 | 设定的安全库存 |
| 缺口数量 | `安全库存 - 可用库存` |
| 日均消耗 | 过去 N 天的平均日出库量 |
| 预计断货天数 | `可用库存 / 日均消耗` |
| 建议采购量 | 基于历史消耗和补货周期计算 |
| 推荐供应商 | 该物料的首选供应商（来自供应商-物料关联） |
| 参考单价 | 首选供应商的最近采购单价 |
| 参考币种 | 首选供应商的结算币种 |
| 预计金额 | `建议采购量 × 参考单价` |

#### 3.11.2 补货策略配置

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| 历史分析天数 | 计算日均消耗的天数范围 | 90 天 |
| 补货周期(天) | 从下单到到货的天数 | 7 天 |
| 安全天数 | 额外预留的安全天数 | 3 天 |
| 批量倍数 | 采购数量对齐到的最小倍数 | 1 |

**建议采购量计算公式**：

``` 
建议采购量 = (日均消耗 × (补货周期 + 安全天数)) - 可用库存 + 安全库存
如果 批量倍数 > 1:
    建议采购量 = CEIL(建议采购量 / 批量倍数) × 批量倍数
```

#### 3.11.3 功能要求

- **补货看板**：集中展示所有需要补货的物料，按紧急程度（预计断货天数）排序
- **一键生成采购单**：勾选物料后一键创建采购单（自动填入首选供应商、物料、数量、单价）
- **消耗趋势图**：展示各物料的近 N 天日出库趋势图，辅助补货决策
- **交货周期管理**：从供应商-物料关联中读取交货周期，精确计算补货时点
- **补货历史记录**：记录每次补货建议的生成和实际采购情况
- 支持导出补货建议为 Excel

- 应用内检查更新（可选）

### 4.6 数据库迁移预留（v1.0 → v2.0）

v2.0 计划从 SQLite 迁移至 PostgreSQL，v1.0 在架构设计上需做好以下预留：

- **Repository 抽象层**：Rust 端所有数据访问操作通过 trait 抽象，Service 层不直接拼接 SQL，便于后续替换底层数据库实现
- **SQL 方言隔离**：将 SQLite 特有语法（PRAGMA、datetime 函数、GENERATED 列等）集中在 adapter 模块中，不散布到业务逻辑层
- **ID 生成策略**：v1.0 使用 `INTEGER PRIMARY KEY AUTOINCREMENT`，代码中预留切换为 PostgreSQL `BIGSERIAL` 或应用层雪花 ID 的接口
- **时间戳处理**：统一使用 UTC 存储，应用层格式化为本地时间，避免 SQLite `datetime()` 函数依赖
- **数据迁移工具**：v2.0 提供 SQLite → PostgreSQL 的一键数据迁移脚本，基于 v1.0 的 schema 版本号自动适配

## 5. 术语表
