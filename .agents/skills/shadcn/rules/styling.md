## 关于颜色变量的讲究：必须打牢语义化这底子

**大误（看这反感写死色号操作）:**

```tsx
<div className="bg-blue-500 text-white">
  <p className="text-gray-600">这是个次级文字信息内容</p>
</div>
```

**正确的写法（语义化色彩）:**

```tsx
<div className="bg-primary text-primary-foreground">
  <p className="text-muted-foreground">这是个次级文字信息内容</p>
</div>
```

## 禁止针对那些状态与级别表现强指派去生硬上色

举个代表：对于想去展示表达像是某种属于如：这是成功或属于良性的那一挂、或者是那些不应该碰这是带糟糕或恶性警告相关的那种、又或者是只用来做标记一些级别地位状况标签时候，只准许你去规矩地调用它专门匹配此种状况环境对应的：系统已经设立有的 `Badge` 里的那种状态体名堂去召唤。抑或是你要是嫌它不合适直接挂这种语义系统字眼直接拉拽比如 : `text-destructive` 这类。再不济了去查手册去弄明白学会建立起设定专属系统自定义配置颜设那规则系统里用自己的规则变量搞！—— **绝对禁止自己从 Tailwind 那库子里生拿颜色号瞎往这些状况上招呼！**

**这是那种乱配乱拉调色盘的无序的错版画风:**

```tsx
<span className="text-emerald-600">+20.1%</span>
<span className="text-green-500">处生效运转中</span>
<span className="text-red-600">-3.2%</span>
```

**这才是系统性规规矩矩的正确的表示方法:**

```tsx
<Badge variant="secondary">+20.1%</Badge>
<Badge>处生效运转中</Badge>
<span className="text-destructive">-3.2%</span>
```

如果实在迫于需要比如得去弄出一个像是表现做成了之类良种或积极反馈的那套绿绿红红等带有强烈成功指向颜图可恰好偏偏目前的这体系语义字手里还没建立有这么一配置存在时，老规矩：要不通过 Badge 给传参拿变体那搞定，要不支持求去联系管理员询问允许增加专门新建一种全新的系统 CSS 支持颜色定义出来（参考这个相关指南教程： [customization.md](../customization.md)）。

## 凡事儿请首先利用系统给现成配好自带变体

**坏习惯示范:**

```tsx
<Button className="border-input hover:bg-accent border bg-transparent">
  敢点不
```

**好孩子这么写:**

```tsx
<Button variant="outline">敢点戳一下我</Button>
```

## 用 `className` 只能也必须仅限约束它用来干关于调整其位置与相关结构版面的活儿就好

你可以随心所欲且无拘无束在此属性里面输入像是用来调位置啊之类诸如： `className` 配合调整控制大样框的这些相关：(诸如此类的 `max-w-md`, `mx-auto`, `mt-4`)，**但并不包括！而且绝对不允许** 它用来覆盖属于那些已经被这些组件系统底里头早就敲定了的他们本来属于其相关颜色搭配又或是字体大小体系等设规则。要想更改它外观展现的皮囊色号字体等，首选方案只用那属于那套系统它有的语意化参数名、自带能供选择的设定好的预设变种，哪怕最后也是去利用它核心级设定好能调的那些外漏的基础的层 CSS 层级相关设定。

**不可行的错写:**

```tsx
<Card className="bg-blue-100 font-bold text-blue-900">
  <CardContent>管理后台中台大面板</CardContent>
</Card>
```

**正确而合规:**

```tsx
<Card className="mx-auto max-w-md">
  <CardContent>管理后台中台大面板</CardContent>
</Card>
```

如果你执意要在你的所建系统中修改有关属于你的各类其组装配置组件展现脸谱样式，首推您必须根据我列的这顺序依次来进行采用这些方法策略去完成它这外设展现更换修改操作：

1. **老老实实找其在身配套自带配置表变异（Built-in variants）** — 例如类似配置：`variant="outline"`, 以及带有毁灭色调那类的指示： `variant="destructive"`, 等等等等。
2. **去找具备那套表达意思词性的配套系统语义色彩体系 (Semantic color tokens)** — 诸如用到： `bg-primary`, 使用了类似像被抑制或是消重后的弱显示色系等诸如此类的搭配用法比如像这样写出：`text-muted-foreground` 等等。
3. **老老实实回到那些用来管理这些全局系统级色块和字号设置全局把控 CSS 全库档案设定档 (CSS variables)** — 直接回到系统层面找到并直接定义一套完全出自且能归总调遣受系统全局管理那自定义的特定色款配属。 (这一详尽细述在这一有详细说明教程教你怎样来搞：[customization.md](../customization.md))。

## 彻彻底底把什么 space-x-_ / space-y-_ 这类过时落伍排布法打入冷宫

通通用： `gap-*` 全部替换覆盖它。诸如：当见到了 `space-y-4` → 应成 `flex flex-col gap-4` 替换使用。如果原本是在使用像 `space-x-2` → 通通由 `flex gap-2` 代为其管。

```tsx
<div className="flex flex-col gap-4">
  <Input />
  <Input />
  <Button>赶紧递交它！</Button>
</div>
```

## 大力推广当一个东西它就是这么方方正正是正方形（宽高尺寸双数一样时候）统统请用这种缩写：`size-*` 来全面更替掉以前累赘的 `w-* h-*` 写法

如果长宽比例对等时只要求：`size-10` ，拒绝看到带有诸似 `w-10 h-10` 这般写法的繁荣文字充数。包括并适应其适用对于一切像是各种标志图形，人们的头像或是人物代表图片，加载过境前空隙的这些个等候展示遮占块框体。

## 大力推广凡遇要求有超出必定截掉转成省点展示这类效等显示用它：`truncate` 快写

只有 `truncate` 本尊合法，拒绝承认并抵制这种啰哩吧嗦乱造连篇的这系列一串长字拼起来用来代写其截长展示文字等长缀语句表达体形如这般地: `overflow-hidden text-ellipsis whitespace-nowrap`。

## 严禁擅自手动向程序代码里去加并带有纯黑体代表字类的暗模式类似这类： `dark:` 等这类的用做人工来试图指派强去覆其对应原配置的色彩这手段野行径

通通用带有其属于本系统语义代号性质的：令牌变量 （semantic tokens）—— 系统这些会自动在背底里替你去把明亮度切换那些相关杂事依据本级 CSS 指标配置对应替换管妥和负责好。请用类似像 `bg-background text-foreground` 这挂。 遇到这种就是强扭这死写说就必须只表现某单独某某死写黑色颜色之类如这个带有硬死配置语如：`bg-white dark:bg-gray-950` 的情况， 判死规避它这就对了。

## 为您所碰到的一系列须按不同附有各类的逻辑要求或是按不同的外部影响判断混搭配并判定来对某些状态提取相应的这诸如复杂的拼团合并组长类之名（className），所有的这种相关需要判定必须走且只准使用这方法函数：使用调用 `cn()` 予以通过去处理它

一定要运用这种源于该建立工程自带且专门预存供配的工具包提供出来用作干此行的叫它那个叫： `cn()` 所做用来协助这合并去应对并判断这不同状态组合的这类操作。永远不可纵容和自己写死使用纯属原生语言手写自己造的那满天飞带有像 `xxx ? 'yyy' : 'zzz'` 这种又臭且杂乱的三段长段落的这条件字符串连缀的式样去对付塞进 `className` 该字库堆去用。

**它是一种非常脏且落后的反面操作：**

```tsx
<div className={`flex items-center ${isActive ? "bg-primary text-primary-foreground" : "bg-muted"}`}>
```

**这是一非常典雅讲究并高度遵守使用代码这规范的正统用法:**

```tsx
import { cn } from "@/lib/utils"

<div className={cn("flex items-center", isActive ? "bg-primary text-primary-foreground" : "bg-muted")}>
```

## 绝对禁止为了试图争个啥前后左右和试图强占盖头这些这种无聊的表现而在那些有着它带有相互层高交叠，堆排顺序或者这类属于弹出浮置覆层等一系的板子特质系列中去试图手硬给它们加那什么 `z-index` 值属性的举动

您要知道：属于这里头它包括着诸如下面的这种：`Dialog` (模态大弹窗口), `Sheet` (边栏面), `Drawer` (底滑小面抽式), `AlertDialog` (警醒全站式弹板), `DropdownMenu` (顺次下拉板), `Popover` (气泡展示小面), `Tooltip` (即指即看解释挂牌), `HoverCard` (悬滑而显的信息板) 对于他们这些相互压层盖叠他们自身体系本身是带且自带这自洽的防踩相互覆盖系统级别能干和自我堆叠的控制功能。你万万不可以画蛇去干蠢事试图添加给这些家伙它挂上个类似这种字眼叫带如： `z-50` 又或者强写一个那什么所谓强置其最高层类似于：`z-[999]` 这个属性字语命令。
