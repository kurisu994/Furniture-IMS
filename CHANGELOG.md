## [Unreleased]

### 新增

- **库存管理模块（全栈）**：实现完整的库存查询 → 出入库流水 → 库存盘点 → 库存调拨全流程
  - **库存查询**：后端 2 个 IPC 命令（列表筛选分页 + 详情），列表支持关键词/仓库/分类/预警状态四维筛选，自动计算库存金额和可用量（总量 - 预留量），预警状态（low/high/normal）高亮显示；详情弹窗含分仓汇总、批次明细（含库龄天数）、近 10 笔流水三 Tab 视图
  - **出入库流水**：后端 1 个 IPC 命令（流水列表查询），支持关键词/仓库/变动类型/日期范围多维筛选分页，变动类型覆盖采购入库/销售出库/采购退货/销售退货/盘盈/盘亏/调拨入/调拨出
  - **库存盘点**：后端 5 个 IPC 命令（列表/详情/创建/更新实盘/审核确认），创建时自动快照系统库存，支持整仓或按分类盘点，审核确认时在事务中生成盘盈/盘亏流水并调整库存（复用 `inventory_ops` 的 `increase_inventory`/`decrease_inventory`）
  - **库存调拨**：后端 5 个 IPC 命令（列表/详情/保存/确认/删除），草稿态支持编辑和删除，确认调拨时在事务中完成：源仓减库存 → 目标仓加库存 → 批次搬仓 → 生成调拨出/入双向流水
  - **前端库存查询页面**：列表页（四维筛选 + 预警行高亮 + 库存金额显示）+ 详情 Dialog（分仓汇总 / 批次明细 / 近期流水三 Tab）
  - **前端出入库流水页面**：列表页（多维筛选 + 日期范围 + 变动类型 Badge 颜色区分 + 增减量红绿显示）
  - **前端库存盘点页面**：列表页（新建盘点 Dialog 选仓库/范围/分类）+ 编辑页（实盘录入 + 盈亏差异高亮 + 差异金额计算 + 审核确认）
  - **前端库存调拨页面**：列表页（行内确认/删除操作）+ 编辑页（双仓选择 + 动态物料行增删 + 保存/确认操作）
  - 国际化：inventory/stockMovements/stockChecks/stockTransfers 四个命名空间，覆盖中/越/英三语
- **定制单管理模块（全栈）**：实现完整的定制单报价 → 确认预留 → 转销售单全流程
  - **定制单 CRUD**：后端 4 个 IPC 命令（列表筛选分页/详情/新建编辑/删除），编号自动生成 `CO-YYYYMMDD-XXX`，支持三种定制类型（尺寸/材质/全定制）和三级优先级
  - **定制单业务流转**：后端 2 个 IPC 命令（确认/取消），确认时自动根据定制 BOM 明细创建原材料预留（计算含损耗量），取消时释放预留并恢复库存 `reserved_qty`
  - **定制 BOM**：后端 1 个 IPC 命令（从标准 BOM 复制），自动关联 `custom_order_id`，复制明细和标准成本
  - **成本核算**：后端 1 个 IPC 命令，基于定制 BOM 明细 × 物料参考成本价重新计算成本
  - **转销售单**：后端 1 个 IPC 命令，从已确认定制单创建草稿销售单（带入客户/币种/报价金额），自动关联
  - **前端列表页**：5 维筛选（关键词/客户/状态/定制类型/日期范围）+ 状态操作按钮 + BusinessListTableShell 骨架
  - **前端详情/编辑页**：头信息表单 + 配置明细表格（inline 编辑）+ 定制 BOM 管理区 + 原材料预留状态展示 + 报价信息汇总（成本/加价/报价/预估毛利）
- **生产工单模块（全栈）**：实现工单全生命周期：创建 → 领料出库 → 生产中 → 完工入库 → 关闭/取消
  - **工单 CRUD**：后端创建 3 张表（`production_orders`, `production_order_materials`, `production_completions`）并提供列表/详情/新建编辑/删除基础命令，支持关联标准 BOM 或定向从定制单下发
  - **领料与退料**：后端 `pick_materials`/`return_materials` 命令，调用 `inventory_ops.rs`，支持按批次、仓库超领上限（120%）出仓。同时支持消耗定制单的特殊原材料预留库存
  - **完工入库与成本联动**：后端 `complete_production` 命令，支持多次按批完工入库（存入成品仓），完工时自动基于已领料的动态加权成本平均摊入成品库存
  - **状态流转**：前端交互与后端 `start_production`/`finish_production_order`/`cancel_production_order` 命令闭环，实现从草稿到完工各关键环节锁定保护
  - **前端列表/详情页**：新增 `/production-orders` 路由，包含列表页 5 维状态过滤、详情页/执行态 UI 隔离、独立提供领料、退料、完工入库的数据弹窗表单
  - 国际化与集成：支持中/越/英三语（`productionOrders` 命名空间），新增侧边栏 Hammer 图标导航入口
  - 国际化：customOrders 命名空间，覆盖中/越/英三语

## [0.1.3] — 2026-04-20

### 新增

- **销售管理模块（全栈）**：实现完整的销售单 → 销售出库 → 销售退货全流程，与采购模块完全对称
  - **销售单**：后端 6 个 IPC 命令（列表筛选分页/详情/新建编辑/审核/作废/删除），编号自动生成 `SO-YYYYMMDD-XXX`，多币种支持（从客户带出币种和汇率），订单级单仓模式，行折扣 + 整单折扣率叠加计算（行金额 = 数量 × 单价 × (1 - 行折扣率/100)，整单折扣金额 = 合计 × 折扣率/100），审核时信用额度检查（超限提醒不拦截）
  - **销售出库**：后端 3 个 IPC 命令（待出库明细/出库单列表/确认出库），核心事务在一个 SQLite 事务中完成：库存可用性校验（零库存强制拦截）→ 双成本快照固化（标准成本从 BOM、实际成本从库存平均成本）→ 扣减库存和批次库存 → 记录流水 → 费用分摊（中间批次四舍五入 + 最后一笔倒挤法）→ 联动销售单状态 → 生成应收账款
  - **销售退货**：后端 3 个 IPC 命令（可退明细/退货单列表/确认退货），必须关联原出库单，退货数量校验（不超过可退数量），退货成本回调（使用原出库成本快照回增库存，正确计算移动加权平均成本），批次库存回增，应收账款冲减（负数记录）
  - **前端销售单页面**：列表页（5 维筛选 + 按状态渲染操作按钮 + 出库进度显示）+ 编辑页（明细表格 inline 编辑 + 行折扣列 + 整单折扣率 + 金额汇总区 + 通用物料选择器）
  - **前端销售出库页面**：列表页（从销售单出库下拉选择器）+ 执行页（自动带出待出库明细 + 可用库存显示 + 标准成本列 + 库存不足高亮 + 一键确认出库）
  - **前端销售退货页面**：列表页（从出库单退货下拉选择器）+ 执行页（自动带出可退明细 + 退货数量校验 + 退货原因必填 + 一键确认退货）
  - 国际化：sales 命名空间 130+ 翻译键，覆盖中/越/英三语
- **采购管理模块（全栈）**：实现完整的采购单 → 采购入库 → 采购退货全流程
  - **采购单**：后端 7 个 IPC 命令（列表筛选分页/详情/新建编辑/审核/作废/删除/供应商物料快速带入），编号自动生成 `PO-YYYYMMDD-XXX`，多币种支持（从供应商带出币种和汇率），订单级单仓模式，金额自动计算（合计/折扣/运费/其他费用/订单总金额/USD 折算）
  - **采购入库**：后端 3 个 IPC 命令（待入库明细/入库单列表/确认入库），核心事务在一个 SQLite 事务中完成 7 步操作（生成单号 → 费用分摊 → 插入单据 → 更新库存 → 生成批次 → 记录流水 → 联动采购单状态），费用分摊采用中间批次四舍五入 + 最后一笔倒挤法，移动加权平均成本计算，批次追踪物料自动生成 `inventory_lots` 记录
  - **采购退货**：后端 3 个 IPC 命令（可退明细/退货单列表/确认退货），必须关联原入库单，退货数量校验（不超过可退数量），退货成本回调（重新计算移动加权平均成本），批次库存扣减
  - **库存操作基础模块**：新增 `inventory_ops.rs`，提供 9 个可复用函数（`increase_inventory` / `decrease_inventory` / `recalc_avg_cost_after_return` / `create_inventory_lot` / `decrease_lot_inventory` / `record_transaction` / `generate_lot_no` / `generate_transaction_no` / `convert_to_usd_cents`），供后续销售出库、盘点、调拨等模块复用
  - **前端采购单页面**：列表页（6 维筛选 + 按状态渲染操作按钮）+ 编辑页（明细表格 inline 编辑 + 供应商物料快速添加 + 金额汇总区）
  - **前端采购入库页面**：列表页（从采购单入库下拉选择器）+ 执行页（自动带出待入库明细 + 批次号输入 + 110% 溢量校验 + 一键确认入库）
  - **前端采购退货页面**：列表页（从入库单退货下拉选择器）+ 执行页（自动带出可退明细 + 退货数量校验 + 退货原因 + 一键确认退货）
  - 审核/作废采用原子 `UPDATE WHERE status = ?` 模式防止 TOCTOU 竞态
  - 入库确认事务中采购单状态校验在事务内执行，防止并发超量入库
  - 入库超量后端校验：单次入库数量不超过剩余可入库数量的 110%
  - 退货可退数量后端校验：退货数量不超过原入库批次剩余可退数量
  - 国际化：purchase 命名空间 100+ 翻译键，覆盖中/越/英三语
- **公共分页控件**：新增 `components/common/pagination.tsx`，提供 `PaginationControls` 组件和 `buildPageNumbers` 函数，统一所有业务列表页的分页器样式

### 优化

- **业务列表小屏体验**：优化物料和供应商列表在小屏下的横向滚动与固定首列表现
  - 表格本体使用明确最小总宽度和稳定列宽，避免小屏下列被压缩却不能滚动
  - 修复主布局缺少 `min-w-0` 导致子级 `overflow-x-auto` 无法生效的问题
  - 将 sticky 首列背景改为不透明实底色，避免横向滚动时透出后方文字影子

  - 移除 `.prettierrc` 配置文件
  - 新增 `biome.json` 配置，统一单引号、导入排序、尾随逗号等规则
