## 🔴 开发前必须解决的阻塞项（8 项）

### 1. ~~多用户部署架构未明确~~ ✅ 已明确
- **来源**：需求文档 / 数据库设计
- **决策**：v1.0 明确为「单机单帐号」模式（系统仅内置唯一管理员帐号，拥有全部权限），SQLite 本地存储不通过网络共享。v2.0 计划迁移至 PostgreSQL 支持局域网多终端访问。
- **预留措施**：v1.0 开发中通过 Repository trait 抽象数据访问层、SQL 方言隔离在 adapter 模块、迁移脚本预留 PostgreSQL 版本，确保 v2.0 迁移成本可控。

### 2. ~~Next.js 16 版本风险~~ ✅ 已确认
- **来源**：需求文档 / 开发计划
- **决策**：确认使用 Next.js 16 正式版，版本风险已消除。

### 3. ✅ 生产管理模块边界缺失（已解决）
- **来源**：需求文档
- **决策**：v1.0 增加**最简化生产工单模块**，覆盖「创建工单 → 领料出库 → 完工入库」核心流程。工单表（`work_orders` + `work_order_materials`）已加入数据库设计，入库/出库类型增加 `production`，库存流水增加 `production_in`/`production_out`。UI 原型新增工单列表页和详情/执行页。排产计划、工序管理、报工等高级功能推迟至 v2.0。

### 4. ✅ 权限差异化 UI 方案缺失（已解决）
- **来源**：UI 原型设计
- **决策**：v1.0 调整为**单机单帐号**模式，系统仅内置唯一管理员帐号，拥有全部权限，不做角色差异和权限裁剪。权限差异化 UI（管理员/操作员视角差异）推迟至 v2.0 随 PostgreSQL 迁移和多帐号功能一同实现。v1.0 无需补充权限矩阵。

### 5. ✅ 时间戳已统一为 UTC
- **来源**：数据库设计
- **决策**：全部 45 张表的 `created_at` / `updated_at` 已统一使用 `datetime('now')`（UTC）。前端展示时按用户 locale 转换显示。同时补充了应用层 `updated_at` 更新策略说明。

### 6. ✅ `purchase_returns` 索引已补充
- **来源**：数据库设计
- **决策**：已补充 `supplier_id`、`inbound_id`、`status`、`return_date` 4 个索引。同时补充了 `stock_checks`（date/status）、`transfers`（date/status）等表的缺失索引。

### 7. ✅ `reserved_qty` CHECK 约束已添加
- **来源**：数据库设计
- **决策**：`inventory.reserved_qty` 和 `inventory_lots.qty_reserved` 已添加 `CHECK(... >= 0)` 约束，防止应用层 bug 导致预留数量为负。

### 8. ✅ 开发计划缓冲已补充
- **来源**：开发计划
- **决策**：总工期已从 22–24 周调整为 26–28 周。阶段三B联调缓冲从 5d 增至 7d，阶段五回归测试从 4d 增至 6d，UI 打磨从 2d 增至 4d。里程碑时间相应调整。
