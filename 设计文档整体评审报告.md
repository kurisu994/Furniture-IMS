## 二、核心亮点（值得肯定的设计）

### 💰 金额精度全链路设计（业界最佳实践）
- 需求层：原币计算 → 行级舍入 → 汇总，前端不重算，Rust 统一计算
- 数据库层：`INTEGER` 存储最小货币单位整数值，规避浮点精度问题
- UI 层：正确展示多币种格式（₫29,250,000 / ¥280.00 / $1,200.00）
- 四份文档完全一致，从根本上消除了多币种精度隐患

### 🔄 退货冲减模型设计（审计友好）
- 独立调整记录 + `return_offset` 标记，不回写原单
- 数据库 `payables.adjustment_type` / `receivables.adjustment_type` 精确支撑
- 完整的审计追溯链路

### 🇻🇳 越南本地化深度适配
- MST 税号格式校验、省市区三级地址、电话格式
- 中/越/英三语 + VND/CNY/USD 多币种
- 越南语字体回退（Noto Sans Vietnamese）

### 📊 批次追溯与库存预留体系
- 三档追踪模式（不追踪/可选/必填）
- lot 级预留 + FIFO 分配策略
- 二级预留结构，灵活可扩展

### 🎯 v1.0 范围边界清晰
- 全文多处 `v1.0 约定` 标注，有效控制需求蔓延
- 为 v2.0 预留升级路径（多仓、反审核、扫码等）
- 四份文档范围声明完全一致
